Деплой на Render.com (бесплатный веб)
====================================

1. Залейте проект в GitHub (ветка main).
2. Зайдите на https://render.com → New → Blueprint
3. Подключите репозиторий, выберите render.yaml (или создайте Web Service вручную).
4. После первого деплоя откройте Dashboard сервиса → Environment и добавьте:
   - ALLOWED_HOSTS = <имя-сервиса>.onrender.com
   - RENDER_EXTERNAL_HOSTNAME = <имя-сервиса>.onrender.com
     (добавляет хост в ALLOWED_HOSTS и автоматически https://… в CSRF_TRUSTED_ORIGINS — без этого вход даёт 403 CSRF)
   - Render подставляет RENDER=true — включён SECURE_PROXY_SSL_HEADER для HTTPS за прокси
   - Свой домен: дополнительно CSRF_TRUSTED_ORIGINS = https://ваш-домен.ru
5. Для постоянной БД: в Render добавьте PostgreSQL (Free) и подключите к сервису — подставится DATABASE_URL.
   Без Postgres используется SQLite на диске; на free tier данные могут теряться при перезапуске.
6. Создать суперпользователя: в Render → Shell:
   python manage.py createsuperuser
   python manage.py init_roles
7. Депозит ETH (Unit / L1): при ошибке «Нет подключения к Ethereum RPC» задайте в Environment
   свой надёжный endpoint: ETHEREUM_MAINNET_RPC_URL=https://… (Alchemy, Infura, QuickNode и т.д.).

Hyperliquid testnet (локально или в Environment)
===============================================
export HYPERLIQUID_USE_TESTNET=true
Перезапустить приложение. Info API и ордера пойдут на api.hyperliquid-testnet.xyz; в приложении укажите кошелёк и ключ от testnet-аккаунта HL (не mainnet). На дашборде кнопки Mainnet / Testnet слева от «Депозит» сохраняют выбор в сессии (для вашего браузера перекрывает env). Переопределение URL: HYPERLIQUID_INFO_URL=... (legacy) или HYPERLIQUID_MAINNET_INFO_URL / HYPERLIQUID_TESTNET_INFO_URL.

Локальный Gunicorn (как в проде)
================================
cd "hyperliquid prototype"
. .venv/bin/activate
export DEBUG=False
export ALLOWED_HOSTS=127.0.0.1,localhost
gunicorn config.wsgi:application --bind 127.0.0.1:8000

Открыть в браузере: http://127.0.0.1:8000/

Публичный URL через туннель (без Render)
========================================
Установите ngrok или cloudflared и выполните, например:
  cloudflared tunnel --url http://127.0.0.1:8000
После запуска gunicorn в другом терминале.
